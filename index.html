ليه الموضوع ده ضروري لموظف زيك؟

تأسيس الدافع قبل الشرح التقني

في المحاضرة الأولى اتعلمنا إزاي نكتب Prompt صح. النهارده هننقل نقلة كبيرة: هنخلي Claude يشتغل على ملفاتك الفعلية — ملفات Excel اللي فيها فواتير، قراءات عدادات، شكاوى عملاء.

تخيل إنك كل يوم بتفتح ملف فيه 500 صف وبتقعد ساعة تفلتر وتنسخ. النهارده هنشوف إزاي Claude يعملها في 30 ثانية.

المحترف مش اللي بيشتغل أكتر، اللي بيخلّي الأداة تشتغل بدلوه. كل ملف بتفتحه يدوياً = 15 دقيقة. لو 10 ملفات يومياً = 2.5 ساعة ضايعة. في الشهر = 55 ساعة!

٠٣

إزاي Claude بيقرا الملفات + ما هو Table؟

فهم الأساس قبل الرفع — ١٥ دقيقة

Claude مش بس Chatbot — هو محلل بيانات. بيقدر يفتح Excel، يفهم العناوين، يعرف العلاقة بين الأعمدة، ويستخرج اللي إنت عايزه. بس فيه شرط: الملف لازم يكون منظم.

Knowledge
الملفات المرفوعة. لما بترفع ملف على Claude، بيقرأه وبيفهم هيكله. .xlsx أو .csv أو .pdf.
Header Row
صف العناوين. أول صف فيه أسماء الأعمدة. Claude بيفهمه تلقائياً.
Column Reference
الإشارة للعمود. لما تقول "عمود المبلغ"، Claude بيفهم العمود اللي عنوانه "المبلغ".

⚡ Range vs Table — الفرق اللي بيغير كل حاجة

Range ❌

خلايا عادية

  • خلايا بدون هيكل محدد
  • الـ Flow مش هيعرف يضيف Row
  • الأسماء مش معرّفة رسمياً
  • مش مناسب للأتمتة
Table ✅

جدول منظم

  • هيكل محدد بأسماء أعمدة
  • الـ Flow يقدر يضيف Rows
  • كل عمود ليه اسم واضح
  • ضروري للأتمتة والـ Flows

إزاي تحوّل Range لـ Table (خطوة بخطوة):

  1. افتح الملف في Excel Online (مش Desktop)
  2. حدد كل البيانات (Ctrl+A)
  3. Insert → Table → علّم على My table has headers
  4. سمّي الجدول: ComplaintsTable (من Table Design)
  5. احفظ الملف على OneDrive
تحذير مهم

لو Excel مش Table، الـ Flow مش هيلاقي مكان يضيف فيه Row. تحويل Excel لـ Table هو أهم خطوة في الإعداد.

⚠️ Hallucination + أمان البيانات

مفاهيم حرجة قبل التعامل مع الملفات — ١٠ دقايق
Hallucination مع الأرقام

Claude ممكن يخطئ في قراءة الأرقام الكبيرة أو يحسب حسابات غلط. لو الملف فيه 1000 صف، ممكن يخلط بين صفوف.

الحل: دايماً راجع الأرقام المهمة. استخدم Claude للاستخراج والتصنيف، بس الحسابات الحساسة راجعها في Excel.

🔒 أمان البيانات مع الملفات

❌ مترفعش:
- بيانات عملاء حقيقية (أسماء + تليفونات + عناوين)
- أرقام بطاقات أو حسابات بنكية
- معلومات سرية للشركة أو رواتب الموظفين
- كلمات مرور أو بيانات دخول

✅ Checklist قبل الرفع:
☐ استبدل الأسماء ببيانات تجريبية
☐ احذف الأعمدة الحساسة
☐ اعمل نسخة احتياطية (Backup)
☐ حوّل الملف لـ Table
☐ تأكد إن صف العناوين واضح

القاعدة: أي ملف هترفعه، اسأل نفسك: "لو الملف ده وصل لحد مش المفروض يشوفه، فيه مشكلة؟" لو أيوه — نظّف الملف الأول.

٠٥

جملة الأمان + Demo: استخراج وتعديل

القلب الحقيقي للمحاضرة — ٢٠ دقيقة

ليه المحور ده هو الأهم؟ لأنك بتتعامل مع بيانات حقيقية. لو طلبك غامض، Claude ممكن يعدّل أعمدة إنت مش قاصدها.

الخطر الحقيقي

لو كتبت "نضّف الملف ده" — Claude ممكن يحذف صفوف، يغيّر أعمدة، أو يضيف بيانات من عنده. في ملف فواتير حقيقي، ده ممكن يسبب خسارة مالية.

في كل Prompt بتتعامل فيه مع ملف حقيقي، لازم تكتب في الآخر:
"باقي كل الأعمدة والصفوف سيبها زي ما هي بالظبط من غير أي تعديل."

Prompt غامض ❌

من غير حماية

  • "نضّف الملف ده"
  • "عدّل البيانات"
  • "رتّب الملف"
  • النتيجة: Claude ممكن يلمس أي حاجة
Prompt محكم ✅

مع جملة الأمان

  • "في عمود الحالة، غيّر Pending لـ Completed"
  • "استخرج بس الفواتير فوق 5000"
  • "باقي كل الأعمدة سيبها زي ما هي"
  • النتيجة: تعديل دقيق + حماية كاملة
مثال ١

استخراج العملاء المتأخرين

السيناريو: ملف فيه 500 فاتورة. المدير عايز يعرف مين العملاء اللي فاتورتهم فوق 5,000 درهم وحالتهم "متأخر".
[الشخصية] أنت محلل بيانات في شركة كهرباء ومياه. [المطلوب] من الملف المرفق، استخرج كل الصفوف اللي: - المبلغ فيها أكبر من 5,000 درهم - الحالة = "متأخر" [التنسيق] حط النتيجة في جدول: اسم العميل، رقم العداد، المبلغ. [الأمان] باقي الملف متلمسوش. [إضافي] احسب إجمالي المبالغ المتأخرة.
✅ النتيجة: جدول فيه 15-30 عميل + الإجمالي. الوقت: 30 ثانية بدل 25 دقيقة يدوياً.
مثال ٢

تعديل حالة الفواتير بأمان

السيناريو: عمود "الحالة" فيه صفوف "Pending" عايز تغيّرها لـ "Completed".
[الشخصية] أنت محلل بيانات في شركة كهرباء ومياه. [المطلوب] في عمود "الحالة"، أي صف قيمته "Pending" غيّرها لـ "Completed". [الأمان] باقي كل الأعمدة والصفوف سيبها زي ما هي بالظبط من غير أي تعديل. [التنسيق] ارجعلي الملف كامل واعرضلي عدد الصفوف اللي اتغيرت.
✅ النتيجة: Claude يغيّر بس عمود الحالة، ويرجع الملف كامل.
مثال ٣

تقرير ملخص شهري تلقائي

[الشخصية] أنت مدير مالي في شركة كهرباء ومياه. [المطلوب] من ملف الفواتير، اطلع تقرير مختصر فيه: 1. إجمالي المبالغ (مدفوع + متأخر) 2. عدد الفواتير المدفوعة والمتأخرة 3. أعلى 5 عملاء في المبلغ [الأمان] الملف الأصلي متلمسوش.

تمرين عملي: ملفك إنت

التطبيق الفعلي — ٢٠ دقيقة

دلوقتي دورك. هترفع ملف Excel وتكتب Prompt لاستخراج أو تعديل بيانات. استخدم جملة الأمان دايماً.

Plan B — لو ما عندكش ملف جاهز

اعمل ملف Excel جديد فيه 10 صفوف:
- الأعمدة: الاسم | رقم العداد | المبلغ | الحالة
- حط 3 صفوف حالتهم "Pending" و 7 "Completed"
- حوّله لـ Table وسمّيه: TestTable
- ارفعه على Claude

تدريب ١ — استخراج بيانات

٧ دقايق
  1. ارفع الملف على Claude
  2. اكتب Prompt يستخرج بيانات محددة (مثلاً: صفوف فوق مبلغ معين)
  3. تأكد إن جملة الأمان موجودة
  4. شوف النتيجة وقارنها بالملف الأصلي

تدريب ٢ — تعديل بيانات

٧ دقايق
  1. اكتب Prompt يعدّل عمود معين
  2. تأكد إن جملة الأمان موجودة
  3. شوف النتيجة: هل التعديل حصل بس في العمود المطلوب؟
  4. قارن الملف قبل وبعد

تدريب ٣ — تقرير ملخص

٦ دقايق
  1. اطلب من Claude يطلع تقرير ملخص من الملف
  2. راجع الأرقام: هل دقيقة؟
  3. تذكّر: الأرقام الحساسة لازم تتراجع يدوياً

التدريب الختامي والواجب

تطبيق شامل — آخر ١٠ دقايق

تدريب ختامي — ٣ مهام على ملف حقيقي

١٥ دقيقة
  1. افتح ملف Excel من شغلك وارفعه على Claude
  2. المهمة ١: استخراج بيانات محددة (فواتير فوق مبلغ معين)
  3. المهمة ٢: تعديل عمود مع جملة الأمان
  4. المهمة ٣: تقرير ملخص من الملف
  5. راجع الأرقام — تأكد إن مفيش Hallucination
  6. احسب الوقت اللي وفرته

الواجب البيتي — سلّميه في المحاضرة الجاية

  • جهّز نسخة تجريبية من ملف Excel بتشتغل عليه يومياً (نظّف البيانات الحساسة)
  • حوّله لـ Table واحفظه على OneDrive
  • طوّر الـ Project بـ 3 Skills ثابتة لمهامك اليومية
  • جرّب 3 قوالب من المرجع السريع على ملفات حقيقية
  • احسب الوقت اللي وفرته في كل مهمة (دقايق × مرات التكرار)
